How FraudShield Works
A complete end-to-end ML pipeline — from raw data to live fraud prediction with AI explanations.
01
📂

Data Loaded

284,807 real credit card transactions with 30 features (V1–V28, Time, Amount) — zero missing values.

02
⚙️

10 Models Trained

Benchmarked RF, XGBoost, CatBoost, AdaBoost, LightGBM, SVM, KNN, LR, DT, and GB in parallel.

03
🔧

Tuned & Selected

GridSearchCV with 3-fold CV. Best params: depth=20, min_split=5, estimators=100. Random Forest deployed.

04
🚀

Live Deployment

Django + Groq API (LLaMA 3.1-8b-instant) serve predictions and AI explanations on PythonAnywhere.

🐍 Python ML Stack

Scikit-learn, CatBoost, XGBoost, LightGBM, Pandas, NumPy, Matplotlib, Seaborn. Full GridSearchCV pipeline with stratified splits.

🌐 Django Backend

Django REST framework hosts the prediction API. Trained model persisted as fraud_model.pkl via joblib for instant inference.

🤖 Groq + LLaMA 3.1

Groq API with llama-3.1-8b-instant generates real-time natural language fraud explanations with risk factor breakdowns.

🗃️ Credit Card Dataset

Kaggle dataset: 284,807 transactions, 492 fraud (0.17%). Features V1–V28 are PCA-transformed for privacy. Time and Amount retained as-is.

⚖️ Imbalance Handling

Severe class imbalance managed through stratified splits and metric prioritisation — F1-Score and AUC-ROC over raw accuracy.

☁️ Live Deployment

Hosted on PythonAnywhere at fraudsystem.pythonanywhere.com. SQLite for session-based transaction history storage.
